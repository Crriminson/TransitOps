A. Architecture and Code Quality

Build a scalable solution

Keep the code well-structured

Maintain clear, readable code

Follow good coding standards

Use modular architecture and sensible coding patterns

Understand the tools and stack you use

B. Database Design

Give database design high priority

Model data properly

Use a local database such as MySQL or PostgreSQL

Design data flow carefully to support scalability and maintainability

C. Backend and APIs

Design clean backend APIs

Prefer building core features yourself instead of relying on Backend-as-a-Service

Avoid unnecessary third-party API dependence

Keep the backend self-contained where possible

D. Data Handling

Use real-time and dynamic data for the final solution

Static JSON can be used only for prototyping

Final product should not depend on hardcoded/static data

E. Validation and Error Handling

Implement robust input validation

Handle user errors gracefully

Show clear feedback for invalid input, like wrong email or bad form data

F. Frontend and UX

Build an interactive, clean UI

Keep a consistent color scheme

Ensure intuitive navigation

Focus on usability and smooth user flow

Make the frontend polished and easy to use

G. Version Control and Teamwork

Use GitHub properly

Maintain good version control

Ensure all team members contribute

Presentation should be done by all teammates

H. Technology Choices

Using trendy tech is fine only if it adds real value

Do not use tools just because they are popular

Choose technologies that fit the problem well

I. Evaluation Focus

You will be judged on:

Coding standards

Logic

Modularity

Frontend design

Performance

Scalability

Security

Usability

Debugging skills

Database design

Problem-solving approach
